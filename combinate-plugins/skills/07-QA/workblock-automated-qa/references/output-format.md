# Output format

The skill produces three layers of output:

1. **Per-suite raw data** вЂ” JSON, screenshots, Lighthouse HTML reports
2. **Aggregated report** вЂ” single self-contained `report.html`
3. **Summary** вЂ” `summary.md` and `summary.json` for the Teamwork comment

All three live under `{outputDir}/`.

## `report.html` вЂ” the single-file dashboard

A self-contained HTML report with:

- Header: project, workblock, verdict (large badge), run time
- Suite grid: one card per suite, colour-coded by verdict, expandable for details
- Page matrix: sortable HTML table вЂ” rows are pages, columns are suites, cells are coloured PASS/WARN/FAIL
- Issues list: filterable by severity / suite / page; each issue links to source artefact
- Lighthouse panel: embedded score gauges per page, links to per-page Lighthouse HTML reports
- Screenshots gallery: thumbnail grid, click to expand
- Config panel: the full `qa-config.json` used, in case the reader needs to know what was tested

All assets inline (CSS, no external JS). The file is fully portable вЂ” zip and email if needed.

## Verdict badges (colour codes)

- `PASS` вЂ” green (`#1a8917`)
- `WARN` вЂ” amber (`#c97a00`)
- `FAIL` вЂ” red (`#c0292a`)
- `n/a` вЂ” grey (`#666`)

Use the same palette in `report.html` and `summary.md` (Markdown badges via shields.io or inline emoji вљЄрџџўрџџЎрџ”ґ if using Slack/markdown that doesn't render images).

## Stable filenames

Filenames are deterministic so reruns overwrite cleanly:

- Suite outputs: `{outputDir}/{suite}/{name}.{ext}`
- Aggregated: `{outputDir}/report.html`, `summary.md`, `summary.json`

Don't suffix with timestamps in the file path вЂ” the timestamp lives inside the JSON. If the user wants run history, `outputDir` should be parameterised with the run ID (`/tmp/{project}-qa/{workblock}/{run-id}`).
